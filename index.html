Intercepting Android App APIs

Using Waydroid & Burp Suite on Linux | A comprehensive guide for mobile security analysts and reverse engineers.

Step 1: Install Waydroid

First, install the Waydroid container on your Linux host. If you are using Fedora, you can easily install it via the default package manager:

sudo dnf install waydroid
Note: If you are using a different Linux distribution, please refer to the official Waydroid Desktop Installation Guide.

Step 2: Install Burp Suite

Burp Suite will act as our proxy to intercept HTTP/HTTPS traffic.

  1. Download the Linux x64 installer from the Official PortSwigger Website.
  2. Open your terminal, navigate to your downloads folder, and make the script executable: 
    sudo chmod +x burpsuite_community_linux_v*.sh
./burpsuite_community_linux_v*.sh
  3. Follow the graphical installer prompts (click "Next" until finished).

Step 3: Enhance Waydroid (Root & ARM Translation)

To analyze modern apps, we need ARM translation (to run standard APKs on x86 machines) and root access. We will use waydroid_script to automate this.

git clone https://github.com/casualsnek/waydroid_script
cd waydroid_script
python3 -m venv venv
venv/bin/pip install -r requirements.txt
sudo venv/bin/python3 main.py

When the interactive menu appears, select Android 13 and choose the Install option.
(Navigation tip: Use your keyboard arrows, press Space to select an item, and Enter to confirm).

Make sure to select the following packages:

  • microG: For Google Play Services functionality without heavy bloatware.
  • libndk: ARM translation layer.
  • Magisk: To manage root access cleanly.

Step 4: Configure Burp Suite Proxy Settings

We need to configure Burp Suite to listen to traffic coming from Waydroid.

  1. Open Burp Suite and go to the Proxy tab → Proxy settings.
  2. Under Proxy listeners, add or edit a listener.
  3. Bind the port to 8080 (or 8000).
  4. Crucial Step: For the "Bind to address" option, choose All interfaces or select your specific local network IP (e.g., 192.168.x.x). Do not leave it as 127.0.0.1, otherwise, Waydroid won't be able to reach it.

Step 5: Export & Convert the CA Certificate

To intercept HTTPS traffic, Android needs to trust Burp Suite's Certificate Authority (CA).

  1. In your Linux host browser, configure it temporarily to use the 127.0.0.1:8080 proxy.
  2. Navigate to http://burp and click CA Certificate to download it (usually named cacert.der).
  3. Open your terminal and convert the downloaded .der file to .pem format, then rename it to an Android-compatible system hash: 
    # Convert DER to PEM
openssl x509 -inform DER -in cacert.der -out cacert.pem

# Extract the subject hash
openssl x509 -inform PEM -subject_hash_old -in cacert.pem | head -n 1

    The second command will output a string of characters (e.g., 9a5ba575). Copy this string.

    # Rename the certificate using the hash with a .0 extension
mv cacert.pem <your_hash_string>.0
# Example: mv cacert.pem 9a5ba575.0

Step 6: Route Waydroid Traffic Through Burp Suite

Now, start Waydroid and connect ADB.

waydroid session start
waydroid adb connect

(If a prompt appears in Waydroid asking to "Allow USB debugging", check "Always allow" and click OK).

Apply the global proxy to Waydroid using your Linux host's local IP address (find this using the ip a command):

waydroid shell settings put global http_proxy <YOUR_HOST_IP>:<BURP_PORT>

Step 7: Inject the CA Certificate as a System Cert

Modern Android versions (Android 8.0+) ignore user-installed certificates for apps. To analyze API signatures properly, we must inject our Burp cert directly into the system partition.

  1. Push your hashed certificate to Waydroid's download folder: 
    adb push <your_hash_string>.0 /sdcard/Download/
  2. Open the Waydroid root shell and remount the system partition as read-write: 
    adb shell
su
mount -o remount,rw /
  3. Move the certificate to the system CA directory and apply the correct permissions: 
    cp /sdcard/Download/<your_hash_string>.0 /system/etc/security/cacerts/
chmod 644 /system/etc/security/cacerts/<your_hash_string>.0
  4. Reboot Waydroid to apply changes.

Step 8: Install Apps and Start Analyzing!

You are now ready to intercept APIs. You can easily install target applications via terminal:

# For a standard APK:
waydroid app install "app_name.apk"

# For split APKs (XAPK extracted):
adb install-multiple base.apk split_config.arm64_v8a.apk split_config.xxhdpi.apk

Open the app inside Waydroid. You will now see all backend API calls, requests, and responses flowing directly into your Burp Suite dashboard!
